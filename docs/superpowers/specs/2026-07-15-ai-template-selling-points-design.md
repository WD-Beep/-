# AI 话术模板与链接卖点设计

## 目标

在不改变现有批量邮件审核、发送和权限规则的前提下，让业务员在 AI 批量生成前选择并维护话术模板；链接库则以“人工卖点 + 链接自动提取卖点”共同约束生成内容。部署后使用真实 DeepSeek、真实链接和真实红人完成端到端验证。

## 现状与根因

- 批量邮件页面已经支持逐封预览、编辑、审核、重生成和分页，但 `copyMode === "ai"` 创建活动时没有传 `message_template_id`，AI 只能从候选话术中自行选择，无法严格按业务员指定模板生成。
- `MessageTemplate` 已有完整 CRUD，但只有正文、语言、平台等基础字段，没有生成规则、默认模板和上传解析能力。
- 链接库已自动提取 `extracted_knowledge.selling_points`，页面仍以“摘要”和原始 JSON 为主；人工卖点没有独立字段，无法表达人工优先级。
- 链接生成器只读取自动提取知识，未把人工卖点和选定话术模板作为明确约束传给模型。

## 数据设计

### MessageTemplate

新增：

- `generation_rules` JSONB，保存 `tone`、`language`、`min_length`、`max_length`、`subject_format`、`body_structure`、`required_content`、`forbidden_content`、`cta`。
- `is_default` Boolean，同一产品最多一个业务默认模板；系统种子模板仍保留 `is_system_default` 只读标识。
- `source_filename` String(255)，记录上传来源文件名，不保存原文件。

### LinkKnowledgeBase

新增：

- `manual_selling_points` JSONB 字符串数组。人工卖点与自动提取卖点分开保存，重新抓取不会覆盖人工内容。

## 后端行为

### 模板维护

- 现有模板 CRUD 扩展新字段。
- 创建或更新 `is_default=true` 时，在同一产品事务内清除其他业务默认模板。
- 删除默认模板后不自动删除或修改其他业务数据；AI 页面回退到系统默认或列表首项。
- 新增模板文件解析接口，支持 UTF-8/GB18030 的 `.txt`、`.md`，以及从 `.docx` 的 `word/document.xml` 提取纯文本；限制文件大小和扩展名。

### 批量 AI 邮件

- AI 模式创建活动时也保存 `message_template_id`。
- AI 预览必须加载指定模板；未指定时选择该产品业务默认模板，其次系统默认模板，再次列表首项。
- 生成 prompt 明确包含模板正文和 `generation_rules`，模板作为结构与语气框架，红人资料和品牌知识用于个性化填充。
- 输出校验标题、正文、未替换占位符、长度、必须包含和禁止内容。仅长度或规则不合格时携带校验原因自动重试一次；第二次仍失败则返回结构化失败，不进入发送队列。
- 逐封重新生成沿用活动保存的模板，不覆盖其他已人工编辑邮件。

### 链接库卖点

- 生成输入中的卖点按“人工卖点在前、自动提取卖点在后”去重合并。
- prompt 要求仅从合并后的卖点选择与当前红人匹配的 2–4 条，不允许编造。
- 链接生成请求可携带 `message_template_id`；生成器读取模板正文和规则，并与卖点共同约束输出。
- fallback 也使用合并卖点，确保 AI 不可用时不会丢失人工输入。

## 前端交互

### AI 批量邮件

- “AI 生成”选项下显示话术模板区域：模板选择、默认标识、模板预览、生成规则摘要。
- 支持在当前区域新建、编辑、删除、设默认；复用现有模板 CRUD 和表单组件。
- 支持上传 `.txt`、`.md`、`.docx`，解析后填入新建模板表单，业务员确认后保存。
- AI 模式没有可用模板时阻止生成并给出中文提示。

### 链接库

- 将“摘要”主编辑区改为“产品卖点 / 核心优势”，按行编辑卖点。
- 自动提取卖点只读展示，人工卖点可新增、删除、保存；明确提示人工卖点优先。
- 摘要和结构化 JSON 移入“高级信息”折叠区，保留原有编辑能力。
- 生成区增加话术模板选择，并将模板 ID 传给后端。

## 错误处理与安全

- 不记录 API Key、密码或完整邮件正文。
- 继续使用现有 `/api-long` 长请求代理和结构化 API 错误；模型失败、模板校验失败、链接抓取失败分别显示真实原因。
- 上传文件仅在内存解析，不落盘；限制 2MB，拒绝不支持格式和空内容。

## 验证

- 后端测试覆盖默认模板唯一性、文件解析、AI 模板传递与规则校验、人工卖点优先、链接模板传递。
- 前端测试覆盖 AI 模式携带模板 ID、模板管理入口、上传入口、卖点编辑器和高级信息折叠。
- 运行 API 相关 pytest、Web node tests、`npm.cmd run lint`、`npm.cmd run build`。
- 部署迁移后，通过公网创建模板、录入人工卖点、抓取真实链接、选择真实红人并调用 DeepSeek，检查生成长度、卖点命中、个性化、逐封编辑和重新生成。
