# 后台业务员联系方式与安全删除设计

## 目标

后台管理员创建或编辑业务员时，联系方式允许保存邮箱、手机号、微信号、内部账号或备注文本；管理员可以直接删除业务员，同时解除负责人关系并完整保留历史业务数据。

## 联系方式

- 保留现有数据库列和 API 字段名 `email`，避免影响登录和真实收件人邮箱逻辑。
- `AdminUserCreate.email` 与 `AdminUserUpdate.email` 改为最大 255 字符的可选普通字符串。
- 前端字段统一显示为“邮箱 / 手机 / 联系方式”，移除邮箱正则、`type="email"` 和非邮箱文本丢弃逻辑。
- 登录账号 `username` 的非空、长度和字符规则保持不变。

## 删除事务

删除接口继续要求管理员权限并禁止删除当前登录管理员。事务内执行：

1. 统计品牌、任务、邮件、回复、活动、发送队列等关联数据。
2. 删除 `product_members` 和 `workspace_members` 中账号自身的成员关系。
3. 将采集任务、任务候选、邮件、回复、手动邮件、链接库、邮件活动和发送队列的 `user_id` 设为 `NULL`。
4. 将该业务员的自动发送活动设为 `cancelled` 并关闭自动发送。
5. 将尚未发送的 `queued`、`scheduled`、`sending` 队列项设为 `cancelled`，但保留记录和内容。
6. 写入持久化管理员审计日志。
7. 删除 `users` 账号记录并提交一次事务。

任何步骤异常均回滚，接口返回明确中文错误。

## 数据库约束

- `outreach_email_campaigns.user_id` 改为可空并使用 `ON DELETE SET NULL`。
- `outreach_send_queue.user_id` 改为可空并使用 `ON DELETE SET NULL`。
- 新增 `admin_audit_logs`，保存操作者、目标账号快照、关联数量和操作时间。
- 品牌和工作区成员关系仍属于账号本身，可随账号删除。
- 消息模板属于账号私有配置，保留现有删除策略；已发送邮件、回复、红人、品牌、任务、活动和队列历史不删除。

## 前端交互

- `/admin/users` 操作菜单增加“删除”。
- `/admin/products` 原复杂迁移弹窗改为简单确认弹窗。
- 弹窗展示业务员名称、登录账号、品牌数、任务数，以及红人/邮件/回复等其他历史数据是否存在。
- 删除成功后刷新列表并显示后端返回的解除数量；失败时保留弹窗并显示真实中文原因。

## 公网 AI 故障

线上证据显示链接生成仍通过 30 秒 `/api-proxy`，Next.js 日志出现 `socket hang up` 和 `ECONNRESET`。本地已将链接刷新、批量生成和单条重生成切换到 `/api-long`，部署时重建 Web 镜像，并把 Nginx 长请求超时从 300 秒对齐到 900 秒。
