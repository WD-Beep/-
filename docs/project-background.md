# 海外红人数据采集与 AI 邮件外联系统项目背景说明书

## 1. 项目背景

本项目是一个面向海外红人营销业务的内部系统，主要用于帮助业务团队批量采集海外红人数据、提取联系方式、筛选高价值红人，并通过 AI 生成定制化合作邮件，最终完成自动化邮件触达和后续跟进管理。

在传统红人开发流程中，业务员通常需要人工搜索平台账号、复制主页链接、整理邮箱、判断账号质量、编写合作话术、手动发送邮件并记录回复。这个过程耗时长、重复工作多，而且容易出现重复采集、遗漏有效邮箱、话术不统一、跟进记录分散等问题。

本系统的目标是把这些分散的人工步骤整合成一套连续流程，让业务员只需要创建采集任务、填写关键词和邮件模板要求，系统即可自动完成采集、去重、筛选、AI 生成邮件、进入发送队列、发送记录和回信管理。

## 2. 业务痛点

当前红人开发业务主要存在以下问题：

- 红人来源分散，需要在 Instagram、YouTube、TikTok、Facebook 等平台反复搜索。
- 相同关键词多次采集时容易出现重复数据，导致实际新增红人数量偏少。
- 很多红人主页没有直接邮箱，需要从简介、外链、联系页面等位置进一步提取。
- 人工判断红人是否适合产品比较慢，缺少统一评分标准。
- 业务员手动写邮件耗时长，不同业务员的话术质量不稳定。
- 批量发信后，发送结果、失败原因、回信状态不容易集中管理。
- 数据导出、复盘和后续跟进依赖人工表格，效率较低。

## 3. 项目目标

本项目希望实现以下目标：

- 提高红人采集效率，让业务员可以通过关键词、链接、竞品来源等方式批量获取红人。
- 降低重复采集成本，对已存在红人和已发送对象进行去重，减少无效重复工作。
- 提高有效联系方式获取率，自动提取邮箱、WhatsApp、Telegram、联系页面等信息。
- 统一红人质量判断，通过评分、匹配度、联系方式可信度等指标辅助筛选。
- 提高邮件开发效率，让 AI 根据模板要求和红人资料生成个性化英文邮件。
- 支持采集后自动发信，减少业务员在采集、筛选、写信、发送之间的人工切换。
- 保留完整业务记录，包括采集任务、红人资料、发送日志、回信记录和跟进状态。

## 4. 核心功能

### 4.1 红人数据采集

系统支持通过关键词、平台、国家、类目、链接列表等方式创建采集任务。采集任务运行后，系统会自动获取红人基础资料，包括昵称、主页链接、粉丝量、简介、平台信息、内容方向等。

### 4.2 数据去重与筛选

系统会根据平台账号、主页链接、唯一标识、邮箱和历史记录进行去重。对于重复采集到的红人，系统不会反复插入相同数据，而是尽量保留已有记录并补充新信息。

这也是为什么同一批关键词第二次继续采集时，新增数量可能明显变少：因为第一次已经采到的数据会被识别为重复，第二次只会保留真正新增或补充成功的数据。

### 4.3 联系方式提取

采集完成后，系统会从红人简介、主页链接、Linktree、联系页面等位置提取联系方式，并整理到红人资料中。常见字段包括公开邮箱、商务邮箱、最终可用邮箱、WhatsApp、Telegram、联系页面等。

### 4.4 AI 评分与分析

系统可以结合红人的粉丝量、互动率、内容方向、简介、关键词来源、联系方式完整度等信息进行分析，帮助业务员判断该红人是否适合当前产品推广。

### 4.5 话术库与 AI 邮件生成

业务员可以在创建任务时填写邮件主题模板、正文要求、产品卖点、合作方式和备注。系统不会直接把模板原样发送出去，而是把模板作为 AI 生成邮件的规则和素材。

AI 会根据每个红人的具体信息生成定制化邮件，例如结合红人的内容方向、平台、粉丝情况、简介关键词和产品卖点，生成一封更像人工业务员写出的合作邀约。

### 4.6 自动发送邮件

当任务开启“采集完成后自动 AI 发邮件”后，系统会在采集结束后自动筛选本次新增红人，跳过无邮箱、无效邮箱、重复红人、已发送对象和测试邮箱，再生成邮件并进入发送队列。

发送过程会受到每日发送量、每小时发送量、SMTP/Gmail 配置等限制，避免一次性发送过多导致邮箱风控。

### 4.7 邮件记录与回信管理

系统会记录每封邮件的生成内容、发送状态、失败原因、发送时间和对应红人。后续如有回信，可以在回信中心或邮件记录中查看，方便业务员继续跟进合作。

### 4.8 数据导出

系统支持按照平台、国家、类目、关键词、是否有邮箱、是否可联系、是否高匹配等条件筛选红人，并导出 Excel 文件，方便业务复盘、线下处理或交付其他团队。

## 5. 业务流程

标准业务流程如下：

1. 业务员创建采集任务。
2. 填写平台、关键词、国家、类目、采集数量等条件。
3. 填写邮件模板要求、产品卖点、合作方式和备注。
4. 系统开始采集红人数据。
5. 系统自动去重，过滤已存在或无效数据。
6. 系统提取邮箱和其他联系方式。
7. 系统根据质量规则筛选可触达红人。
8. AI 根据模板要求和红人信息生成定制化邮件。
9. 邮件进入发送队列，并按限制分批发送。
10. 系统记录发送结果、失败原因和回信状态。
11. 业务员在后台查看数据、导出表格或继续跟进回信。

## 6. 技术架构

项目采用前后端分离架构：

| 模块 | 技术 |
| --- | --- |
| 前端 | Next.js、React、TypeScript、Tailwind CSS |
| 后端 | FastAPI、SQLAlchemy、Alembic |
| 数据库 | PostgreSQL |
| 任务调度 | APScheduler |
| 数据采集 | 平台采集服务、Apify、API Direct 等 |
| AI 能力 | OpenAI 兼容接口、Kimi/DeepSeek 等模型配置 |
| 邮件发送 | SMTP、Gmail/企业邮箱发信配置 |
| 部署 | Docker Compose |

前端主要提供业务操作后台，包括采集任务、红人列表、邮件模板、发送队列、邮件记录、回信中心、设置和管理员功能。

后端主要负责采集任务执行、数据入库、去重、联系方式提取、AI 分析、邮件生成、邮件发送、回信处理和权限隔离。

## 7. 适用场景

本系统适用于以下业务场景：

- Amazon 产品红人推广。
- 海外达人合作邀约。
- 样品寄送合作。
- Affiliate 联盟营销开发。
- TikTok、Instagram、YouTube、Facebook 等平台红人搜集。
- 批量筛选有邮箱、可联系、高匹配的潜在合作对象。
- 业务员日常开发、主管查看进度、管理员统一管理账号和产品。

## 8. 项目价值

系统上线后，可以为业务团队带来以下价值：

- 减少人工搜索和复制资料的时间。
- 降低重复采集和重复发信的概率。
- 提高有效红人邮箱的整理效率。
- 让不同业务员使用更统一、更稳定的话术规则。
- 通过 AI 定制邮件提高邮件自然度和相关性。
- 集中保存采集、发信、回信和跟进记录。
- 让管理者更容易查看业务产出和任务效果。

## 9. 后续优化方向

后续可以继续优化以下方向：

- 增强多平台采集能力，提高不同平台的数据覆盖率。
- 优化去重逻辑，让同一红人在不同平台、不同链接来源下也能更准确合并。
- 增加采集续跑策略，让同一关键词多轮采集时尽量向新数据扩展。
- 增加邮箱质量检测，降低无效邮箱和退信率。
- 优化 AI 邮件生成规则，让邮件更短、更自然、更贴近不同产品。
- 增加发送频率控制和邮箱健康监控，降低进入垃圾箱和邮箱风控的风险。
- 增加任务效果统计，例如采集数量、有效邮箱数量、发送数量、打开/回复情况等。

## 10. 总结

本项目的核心定位是：帮助海外红人营销团队把“找红人、筛红人、写邮件、发邮件、跟进回复”这一整套流程自动化和系统化。

它不是单纯的数据采集工具，也不是单纯的邮件群发工具，而是围绕红人开发业务搭建的一套完整工作台。业务员只需要提供采集方向和合作话术要求，系统就可以自动完成后续大部分重复工作，从而提升开发效率、减少重复劳动，并让业务数据沉淀到统一平台中。
